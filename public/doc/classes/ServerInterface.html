Class ServerInterface

Uses: Fetch - https://github.com/github/fetch a cross browser polyfill for making REST api calls.
Class defined in: public/javascripts/app/core/ServerInterface.js:9

Makes calls to the Server and receives messages. This class is passed to GameComponents which rely on user interaction that needs to notify the server. GameComponents which require to be notified of incoming server messages, should be passed the GameData Class and listen to signals dispatched from it. When plugging the application to an ElectoTank (or any other) Server, this is the only class which should need to change.

gameData <G.GameData>
public/javascripts/app/core/ServerInterface.js:33

GameData is effectively the server application model. All data returned by the server is stored in the GameData. GameData may also dispatch signals which we can use to provide hooks to the rest of the application via signalDispatcher.

signalDispatcher <G.SignalDispatcher>
public/javascripts/app/core/ServerInterface.js:22

Signal Dispatcher is the core communication component of the application. Ensure it is passed into any Class which might need to talk to another. In this context we can listen to gameData signals, and have the signalDispatcher dispatch a signal which can be listened to by GameComponents which require to be notified when some server data has been received. Eg. ReelsComponent can spin the reels when a spin response is received.
