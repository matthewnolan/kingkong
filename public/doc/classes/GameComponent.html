Class GameComponent

Extends: createjs.Container
Class defined in: public/javascripts/app/components/GameComponent.js:9

Super class of any game component. These game components are always initialised with the game's setup file and a signal dispatcher. This allows all game components (eg. ReelsComponent or WinLinesComponent) to access game setup and dispatch and receive events from other areas of the application

cacheComplete( )
public/javascripts/app/components/GameComponent.js:73

Some Vector drawn assets, and Bitmap sprite animations should be played during app initialisation to ensure they are ready to run at runtime. Call this function once this process is completed, and it will signal to the game that this component's drawing/anim is ready to play. For Vectors, call this after Vector drawing is done. For Bitmap sprites, make sure that it's animations have played once then call this. When adding a component requiring this type of caching, you must add 1 to the Game's initailisedNum. nb. Caching only takes place when setup.json's failSafeDelay is used.
